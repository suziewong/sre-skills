# 大促稳定性保障体系

## 核心目标

| 目标 | 指标 | 说明 |
|------|------|------|
| 快速发现 | MTTD < 5min | 从故障到告警 < 5分钟 |
| 快速止血 | MTTR < 15min | 从告警到恢复 < 15分钟 |
| 平稳度峰 | 0 P0故障 | 核心链路无故障 |

## 全链路压测

### 为什么必须做全链路压测

```
单系统压测：各服务都OK
  ↓
组装起来：数据库被打满！
```

**单体压测永远不够**，因为：
- 依赖服务的瓶颈会被放大
- 数据库连接池是共享资源
- MQ积压会触发连锁反应

### 压测执行步骤

```
1. 梳理链路
   - 核心交易链路（下单→支付→库存）
   - 非核心链路（搜索→详情→下单）

2. 数据准备
   - 影子库（与生产数据隔离）
   - 压测数据构造（用户、商品、库存）

3. 分级压测
   - 单机单接口
   - 单链路混合
   - 全链路综合

4. 阈值设定
   - RT_P99 < 200ms
   - 成功率 > 99.9%
   - 资源使用率 < 80%
```

### 压测监控指标

| 指标 | 告警阈值 |
|------|----------|
| CPU | > 80% |
| 内存 | > 85% |
| 数据库连接池 | > 70% |
| MQ消费延迟 | > 1s |
| 错误率 | > 0.1% |

## 限流设计

### 多级限流

```
┌─────────────────────────────────────┐
│ L0: 平台级限流（总QPS上限）           │
├─────────────────────────────────────┤
│ L1: 接入层限流（Nginx/API Gateway）   │
├─────────────────────────────────────┤
│ L2: 应用级限流（单机/分布式）          │
├─────────────────────────────────────┤
│ L3: 服务级限流（核心服务保护）         │
└─────────────────────────────────────┘
```

### 限流配置参考

```yaml
# Nginx 限流
limit_req_zone $binary_remote_addr zone=api:10m rate=100r/s;
limit_req zone=api burst=200 nodelay;

# 应用层限流（单机）
- QPS: 1000
- 并发: 500

# 分布式限流（全局）
- 核心接口: 5万QPS
- 非核心接口: 1万QPS
```

### 限流策略选择

| 策略 | 适用场景 | 优点 | 缺点 |
|------|----------|------|------|
| 信号量 | 本地限流 | 零开销 | 不支持分布式 |
| 令牌桶 | 流量整形 | 允许突发 | 实现复杂 |
| 滑动窗口 | 精确计数 | 精确 | 内存占用 |
| 漏桶 | 匀速消费 | 平滑 | 不适合突发 |

## 降级设计

### 降级原则

> **保护核心链路，非核心可牺牲**

### 降级清单

| 降级内容 | 触发条件 | 影响 |
|----------|----------|------|
| 关闭活动页 | 系统过载 | 用户看到优惠减少 |
| 关闭实时库存 | 库存服务不可用 | 可能超卖（事后补偿） |
| 简化下单流程 | 下单链路超时 | 减少操作步骤 |
| 关闭评论/推荐 | 非核心服务 | 内容质量下降 |
| 降级为本地缓存 | Redis不可用 | 数据可能过期 |

### 降级开关实现

```python
# 降级开关（存储在配置中心）
class DegradeSwitch:
    def __init__(self):
        self.switches = config_center.get("degrade_switches", {})

    def is_on(self, feature: str) -> bool:
        return self.switches.get(feature, False)

# 使用示例
if not degrade.is_on("real_time_inventory"):
    # 降级：使用本地缓存
    inventory = local_cache.get(product_id)
else:
    # 正常：查询实时库存
    inventory = redis.get(product_id)
```

## 监控告警

### 核心监控指标

```
交易指标
  ├── 下单转化率（浏览→下单）
  ├── 支付成功率（下单→支付）
  ├── 支付转化率（发起→成功）
  └── 平均下单时长

系统指标
  ├── QPS/并发数
  ├── RT_P50/P90/P99
  ├── 错误率
  └── 资源使用率

中间件指标
  ├── 数据库连接池使用率
  ├── Redis命中率
  ├── MQ消费延迟
  └── 服务调用成功率
```

### 告警策略

```yaml
# P0告警（立即处理）
- 核心链路错误率 > 1%
- 下单成功率 < 99%
- 数据库不可用

# P1告警（15分钟内处理）
- 非核心链路错误率 > 5%
- RT_P99 > 2s
- MQ积压 > 1000

# P2告警（1小时内处理）
- 系统资源使用率 > 80%
- 监控服务延迟告警
```
