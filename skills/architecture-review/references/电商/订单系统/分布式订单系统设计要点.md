# 分布式订单系统设计要点

## 核心设计原则

1. **订单号唯一性** - 分布式环境下订单号必须全局唯一
2. **状态不可逆** - 订单状态只能向前推进，不能回退
3. **最终一致** - 异步场景下允许短暂状态不一致
4. **幂等优先** - 任何操作必须考虑幂等

## 订单号设计

### 方案一：雪花算法（推荐）

```
timestamp(41bit) + machine_id(10bit) + sequence(12bit)
= 18位数字
```

**优点**：趋势递增、唯一、有序
**缺点**：依赖机器时钟

### 方案二：数据库自增 + 前缀

```
P{date}{auto_increment}
例如：P20241121000001
```

**优点**：简单
**缺点**：单库瓶颈、不可分散

### 推荐方案：雪花变种

```python
def generate_order_id():
    # 雪花算法 + 业务前缀
    snowflake = get_snowflake_id()
    return f"DD{int(time.time() * 1000)}{snowflake % 10000:04d}"
    # 格式：DD(订单标识) + 时间戳(13位) + 序列号(4位)
```

## 订单状态机

```
┌─────────┐
│ CREATED │ ← 下单成功
└────┬────┘
     │ 支付成功
     ▼
┌─────────┐
│ PAID    │ ← 已支付
└────┬────┘
     │ 发货
     ▼
┌─────────┐
│ SHIPPED │ ← 已发货
└────┬────┘
     │ 确认收货
     ▼
┌─────────┐
│COMPLETED│ ← 已完成
└─────────┘

[可取消路径]
CREATED/PAID → CANCELLED（用户取消/系统取消）
PAID → REFUNDING → REFUNDED（退款路径）
```

### 状态机实现要点

```python
# 错误的做法：先查状态再更新（并发不安全）
order = db.get(order_id)
if order.status == 'CREATED':
    db.update(order_id, status='PAID')

# 正确的做法：原子更新（乐观锁）
affected = db.execute("""
    UPDATE orders
    SET status = 'PAID', updated_at = NOW()
    WHERE id = %s AND status = 'CREATED'
""", [order_id])
if affected == 0:
    raise OrderStatusError("订单状态不正确或已变更")
```

## 异步订单处理

```
下单请求
  ↓
创建订单（DB，写）
  ↓
发送消息到 MQ
  ↓
立即返回用户（订单创建成功）
  ↓ [异步]
库存服务消费 → 扣减库存
  ↓
支付服务消费 → 等待支付
  ↓
物流服务消费 → 生成物流单
```

**注意**：
- 订单创建和库存扣减是独立的，允许短时间不一致
- 需要有定时任务扫描超时未支付的订单并取消

## 分库分表策略

### 按用户ID分片

```
shard_key = user_id % shard_count
```

**优点**：查询用户订单高效
**缺点**：热点用户数据集中

### 按订单ID分片

```
shard_key = order_id % shard_count
```

**优点**：写入均匀
**缺点**：查用户订单需要跨库

### 推荐：用户ID + 年月复合分片

```sql
-- 按用户ID分库，按年月分表
shard_key = hash(user_id + year_month) % total_shards
```

## 常见问题

| 问题 | 原因 | 解决方案 |
|------|------|----------|
| 重复下单 | 按钮重复点击 | 幂等键 + 前端防抖 |
| 订单状态不一致 | MQ消息丢失/乱序 | 定时任务 + 状态核对 |
| 超时未支付 | 用户放弃 | 定时任务取消 + 库存释放 |
| 库存已售罄但仍能下单 | 库存未实时扣减 | 预占库存机制 |

## 订单查询优化

### 读扩散问题

用户查订单列表，如果按用户分片，查起来很简单。
但如果要做"运营后台查所有订单"，就需要扫所有分片。

**解决方案**：ElasticSearch 同步

```
订单表（MySQL）
    ↓ 实时同步
ElasticSearch
    ↓ 复杂查询
运营后台（条件筛选、分页、聚合）
```
