# SRE 容灾设计手册

## 容灾核心指标

| 指标 | 简称 | 定义 | 目标值 |
|------|------|------|--------|
| 恢复点目标 | RPO | 允许丢失的数据量 | < 1秒 |
| 恢复时间目标 | RTO | 允许的恢复时间 | < 30秒 |

## 容灾等级

| 等级 | RTO | RPO | 成本 | 适用 |
|------|-----|-----|------|------|
| 数据级 | 分钟级 | 分钟级 | 低 | 一般业务 |
| 应用级 | 小时级 | 小时级 | 中 | 重要业务 |
| 业务级 | 分钟级 | 秒级 | 高 | 核心业务 |
| 同城双活 | 秒级 | 0 | 很高 | 金融级 |

## 多活架构

### 同城双活

```
          负载均衡
              │
        ┌─────┴─────┐
        ▼           ▼
   机房A         机房B
   (主)          (备)
   读写          只读/切换
```

**特点**：
- 两机房同地域，网络延迟 < 1ms
- 正常情况下主机房承担全部流量
- 故障时切换到备机房

### 两地三中心

```
华东机房（主）
    │
    ├── 华东备（同步复制）
    │
    └── 华南容灾（异步复制）
```

**数据同步策略**：
```sql
-- 主从半同步复制
SET GLOBAL rpl_semi_sync_master_wait_point = AFTER_SYNC;
```

## 切换决策

### 切换决策树

```
故障发生
    │
    ▼
核心链路是否受影响？
    │
    ├── 否 → 局部降级，不切换
    │
    └── 是 → 是否确认故障？
              │
              ├── 否 → 继续观察，准备切换
              │
              └── 是 → 执行切换
                      │
                      ▼
                  通知相关方
                      │
                      ▼
                  执行DNS/负载切换
                      │
                      ▼
                  验证服务正常
                      │
                      ▼
                  持续监控，准备回滚
```

### 自动切换 vs 手动切换

| 场景 | 建议方式 |
|------|----------|
| 数据库主库宕机 | 自动切换（丢1秒数据可接受） |
| 单个服务实例宕机 | 自动重启 + 负载均衡 |
| 整个机房故障 | 手动切换（需要确认） |
| 网络分区 | 手动切换（避免脑裂） |

## 切换后处理

### 切后检查清单

- [ ] 核心服务健康
- [ ] 数据库写入正常
- [ ] 缓存服务正常
- [ ] 消息队列正常
- [ ] 对账数据一致

### 数据一致性检查

```sql
-- 检查主从延迟
SHOW SLAVE STATUS\G
  Seconds_Behind_Master: 0  -- 0表示无延迟

-- 检查账户余额一致性
SELECT user_id, SUM(amount) as total
FROM account_balance
WHERE update_time > '切换时间点'
GROUP BY user_id
HAVING total != expected_total;
```

## 常见故障切换场景

### 场景1：数据库主库宕机

```
检测：心跳超时（3次 × 10s）
     ↓
选主：新主库自动提升
     ↓
切换：VIP/域名切换到新主库
     ↓
验证：读写测试
     ↓
通知：发送切换通知
```

### 场景2：机房级故障

```
检测：多AZ探活失败
     ↓
确认：联系机房确认故障
     ↓
决策：启动跨机房切换
     ↓
DNS切换：主域名指向备用机房IP
     ↓
Warm Up：逐步放量，不要瞬间切满
     ↓
观察：监控核心指标
```

### 场景3：网络抖动

```
检测：部分请求超时
     ↓
判断：是否核心链路受影响？
     │
     ├── 否 → 降级非核心，继续观察
     │
     └── 是 → 是否持续恶化？
              │
              ├── 否 → 继续观察
              │
              └── 是 → 考虑切换
```

## 容量规划

### 容量计算公式

```
所需容量 = 基准容量 × 峰值系数 × 安全系数

示例：
- 基准QPS: 10000
- 峰值系数: 3 (大促)
- 安全系数: 1.2 (预留20%)
- 所需QPS: 10000 × 3 × 1.2 = 36000
```

### 扩容决策

| 指标 | 阈值 | 动作 |
|------|------|------|
| CPU使用率 | > 70% | 触发扩容 |
| 内存使用率 | > 80% | 触发扩容 |
| 连接池使用率 | > 70% | 触发扩容 |
| 错误率 | > 1% | 报警并扩容 |
